**v-show 和 v-if 有哪些区别？**

`v-if` 会在切换过程中对条件块的事件监听器和子组件进行销毁和重建，如果初始条件是false，则什么都不做，直到条件第一次为true时才开始渲染模块。
 `v-show` 只是基于css进行切换，不管初始条件是什么，都会渲染。
 所以，`v-if` 切换的开销更大，而 `v-show` 初始化渲染开销更大，在需要频繁切换，或者切换的部分dom很复杂时，使用 `v-show` 更合适。渲染后很少切换的则使用 `v-if` 更合适。

- 相同点：两者都可用作为条件判断元素是否显示。
- 不同点：
  - 1). v-if后可跟`v-else-if` 和 `v-else`用来进行不同条件下的显示组件的不同可能性，v-show只能作为是否展示633渲染，v-show 无论条件真假总会渲染，只是简单地基于 CSS 进行切换属性display。

- 1.手段：v-if是通过控制dom节点的存在与否来控制元素的显隐；v-show是通过设置DOM元素的display样式，block为显示，none为隐藏；
- 2.编译过程：v-if切换有一个局部编译/卸载的过程，切换过程中合适地销毁和重建内部的事件监听和子组件；v-show只是简单的基于css切换；
- 3.编译条件：v-if是惰性的，如果初始条件为假，则什么也不做；只有在条件第一次变为真时才开始局部编译（编译被缓存？编译被缓存后，然后再切换的时候进行局部卸载); v-show是在任何条件下（首次条件是否为真）都被编译，然后被缓存，而且DOM元素保留；
- 4.性能消耗：v-if有更高的切换消耗；v-show有更高的初始渲染消耗；

**使用场景**

基于以上区别，因此，如果需要非常频繁地切换，则使用 v-show 较好；如果在运行时条件很少改变，则使用 v-if 较好。

**总结**

- 相同点：两者都可用作为条件判断元素是否显示。

- 不同点：

   v-if后可跟`v-else-if` 和 `v-else`用来进行不同条件下的显示组件的不同可能性，v-show只能作为是否展示渲染，v-show 无论条件真假总会渲染，只是简单地基于 CSS 进行切换属性display。

v-if判断是否加载，可以减轻服务器的压力，在需要时加载,但有更高的切换开销;v-show调整DOM元素的CSS的dispaly属性，可以使客户端操作更加流畅，但有更高的初始渲染开销。如果需要非常频繁地切换，则使用 v-show 较好；如果在运行时条件很少改变，则使用 v-if 较好。
